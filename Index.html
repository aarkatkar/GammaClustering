Introduction

With the recent explosion in large-scale biological datasets, there is a need for robust models capable of intuitively analyzing them. gClustPath , or Gamma Clustering and Pathway Analysis, presents two models that prioritize ease of interpretation.

  • Gamma clustering: Maximize the likelihood of a null model producing all inter-cluster correlations and no intra-cluster correlations.

  • Gamma pathway analysis: Minimize the likelihood of a null model producing any linking correlation within a pathway.

The math.

The namesakes of Gamma Clustering and Pathway Analysis are two quantities derived from the p-value of a pairwise correlation.

pSig=log(p)pSig=log(p)

pComp=log(1p)pComp=log(1p)

The sum of pSig decreases with the probability of observing all correlations under a null model. The sum of pComp increases with the probability of observing at least one correlation under a null model. pSig and pComp are exponentially distributed variables, and the sum of i.i.d. exponential variables follows a gamma distribution.

Gamma clustering

For each gene ii and cluster jj , gamma-test the pSig and pComp sum to all genes in the cluster.

Ci,j=P(s>(pComp)i,j)Ci,j=P(s>(pComp)i,j)

Si,j=P(s>(pSig)i,j)Si,j=P(s>(pSig)i,j)

In each iteration, the membership probability mm for a gene ii in cluster jj is defined.

mi,j=Ci,jkjSi,kmi,j=Ci,jkjSi,k

Additionally, a void cluster is introduced corresponding to all correlations being null.

mi,void=k=1nSi,kmi,void=k=1nSi,k

The algorithm begins with all genes assigned to a single cluster. The divisive phase ends when the void cluster no longer decreases in size. At this point, the optimization phase begins, where clusters are maximally separated by non-correlation.

mi,j=kjSi,kmi,j=kjSi,k

The optimization phase ends when no genes have assigned to a new cluster.

Gamma pathway analysis

pComp can used as a meaningful dissimilarity measure for a relevant graph algorithm. For pathway analysis, this would be a shortest path algorithm, which finds a pathway connecting two points to minimize the total pComp across all connections:

minPath=argminpathways((pComp))minPath=argminpathways((pComp))
This finds the series of correlations such that a null model would have the smallest likelihood of observing any linking correlation.

Initializing a workflow

The first step in gamma clustering is loading the gClustPath library and the relevant data. Data should be stored in a matrix with rows corresponding to features (i.e., genes) and columns corresponding to samples.

library(gClustPath)
data("foldChange")
dim(foldChange)
#> [1] 932  14

pMatrix

All gamma clustering methods operate on a matrix of p-values. This can be obtained using the pMatrix function. By default, a ranking function is used to force this matrix to follow an exact uniform distribution. This is necessary for gamma clustering, but not gamma pathway analysis. The output argument should be used to switch between uniform (unif) and original (orig) p-values.

pmat <- pMatrix(foldChange, output="orig")
hist(pmat[upper.tri(pmat)], main="Original")


pmat <- pMatrix(foldChange, output="unif")
hist(pmat[upper.tri(pmat)], main="Uniform")

Simple clustering

For simple clustering, use the gammaCluster function. For visualizing these clusters, use gammaPlot

clst <- gammaCluster(pmat)
gammaPlot(pmat, clst$labels)

Recursive clustering

Subsequently, gamma clustering can operate on each individual cluster, producing sub-clusters, before a final optimization step combines correlated clusters. This process can be repeated until clustering converges, or the necessary amount of precision is reached.

rclst <- recGamma(pmat, clst$labels)
rclst2 <- recGamma(pmat, rclst$labels)
gammaPlot(pmat, rclst2$labels)
